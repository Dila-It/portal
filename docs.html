IT-CTS · 系統介紹與使用指南

校園 IT 工作
數位化管理工具組

IT-CTS（IT Collaborative Tool Suite）將日常郵件處理、任務優先排序、大型專案追蹤整合為一套互聯工具，讓 IT 人員從被動回應轉為主動管理。

✓ 免費部署 ✓ 單一帳號登入 ✓ AI 輔助分析
總覽

系統架構總覽

三個子系統各司其職，透過共用帳號與 projectTag 機制形成完整工作閉環。

🏛
IT-CTS Portal
系統導覽入口。登入後查看各系統狀態、公告、布告欄，並可進入系統設定。
dila-it.github.io/portal →
✦
智慧待辦
日常任務執行層。Gmail 整合、Gemini AI 分析、四象限優先排序、今日工作日誌。
dila-it.github.io/todo-app →
📋
專案管理
跨週跨月的里程碑追蹤。總覽儀表板、健康狀態監控、與智慧待辦雙向整合。
dila-it.github.io/project-mgr →
💡
單一帳號，三系統通用。 在 Portal 登入後，直接點擊卡片進入各子系統，無需重新輸入帳號密碼。

子系統 01

IT-CTS Portal

系統登入入口與管理中心。每天從這裡開始，掌握整體狀況。

主要功能

🖥
系統入口卡片
顯示各子系統的上線狀態（線上 / 維護中 / 離線）。智慧待辦卡片即時顯示未完成待辦件數與 Q1 急件數，點擊直接進入。
📢
公告欄
管理員可設定登入後頂部顯示的即時公告，適合發佈停機通知、緊急事項。
📌
布告欄
多筆公告條目，支援置頂釘選。無內容時自動隱藏；有內容時可展開 / 收合，不佔版面。
🎨
UI 主題系統
深色 / 淺色 / 系統跟隨 / 高對比四種模式，加上 Indigo、Blue、Teal、Rose、Amber 五種主色，從設定頁統一調整，即時套用至所有子系統。

系統設定（IT 管理員）

1
進入設定
登入後點擊右上角「⚙ 系統設定」
2
管理各系統狀態
切換上線 / 維護中 / 離線狀態，Portal 卡片即時反映
3
設定公告與布告欄
填入公告文字並啟用；布告欄可新增 / 刪除 / 置頂條目
4
調整 UI 主題
選擇顯示模式與主色，儲存後所有系統同步套用
子系統 02

智慧待辦

以艾森豪矩陣為核心的日常任務管理系統，整合 Gmail 與 AI 分析，幫助 IT 人員在大量郵件中做出正確的優先排序。

艾森豪矩陣 — 四象限

Q1 🔴
立即處理
緊急 × 重要 — 今天就要完成
Q2 🔵
排程規劃
不緊急 × 重要 — 安排時間深度處理
Q3 🟡
快速處理
緊急 × 不重要 — 快速完成或委派
Q4 ⬜
可略過
不緊急 × 不重要 — 可延後或捨棄

主要功能

📧
Gmail 整合同步
連結 Gmail 後一鍵抓取最新郵件，逐封選擇「AI 分析為待辦」或「加入今日工作區」。Gemini AI 自動判斷急迫性、重要性、難易度並給出處理建議。系統會學習記憶你常略過的寄件人，下次自動排除。
⠿
今日工作區
獨立的當日工作日誌，記錄今天實際處理的事項。跨日自動帶入未完成條目（含備註）。支援拖曳合併（桌機用左側把手 ⠿，手機長按 0.4 秒），合併後可「↩ 拉出」還原。
📊
報表匯出
查詢指定日 / 週 / 月的工作紀錄。包含待辦完成清單、日誌條目、拖延分析三個區塊。支援匯出 Excel、CSV 或列印。
🔗
專案整合
待辦可加掛 projectTag 連結至對應專案。四象限卡片顯示所屬專案的里程碑警示（🔴 逾期 / 🟡 即將截止）。

快速上手

1
登入並授權 Gmail
首次使用點擊「📧 Gmail 同步」，授權 Gmail 讀取權限（使用個人 Google 帳號）
2
同步並篩選郵件
系統抓取最新郵件，勾選要處理的項目，選擇動作後點「開始分析」
3
在四象限中處理待辦
AI 分析完成後，待辦自動出現在對應象限。可拖曳調整位置（有確認提示）
4
記錄今日工作
下方今日工作區輸入當天實際處理的事項，✅ 完成 / 📅 明日 / ↗ 轉待辦
子系統 03

專案管理系統

跨週跨月大型專案的里程碑追蹤工具。一頁掌握所有專案健康狀態，與智慧待辦形成「戰略 → 執行」完整閉環。

🔗
與智慧待辦的關係： 專案管理 = 戰略層（里程碑、截止日），智慧待辦 = 執行層（每日任務）。透過 projectTag 欄位連結，里程碑任務可一鍵建立待辦，待辦完成狀況即時反映到專案進度。

主要功能

📊
總覽儀表板
預設首頁。標題列顯示統計摘要（進行中 N / 暫緩 N / 逾期里程碑 N）。每個專案卡片顯示進度條、預期進度白線標記、健康狀態（🟢 正常 / 🟡 即將截止 / 🔴 逾期）及任務完成比例。點擊卡片開啟右側 Drawer。
▶
抽屜式詳情（Drawer）
點擊儀表板卡片，詳情從右側滑入，儀表板常駐後景。可直接點左側 sidebar 切換到其他專案，無需關閉再開啟。Drawer 內可完整編輯專案、里程碑、任務。
🏁
里程碑管理
每個專案可建立多個里程碑（含截止日）。逾期自動顯示警示，提供「📅 延後截止日」或「🔔 建立緊急待辦」快速行動。截止日變更可記錄原因（需求追加 / 人力不足 / 技術難題 / 其他）。
▦
雙視圖：清單 / Kanban
里程碑可切換清單視圖（依序展開）或 Kanban 看板視圖（待開始 / 進行中 / 已完成三欄），依工作習慣選擇。

快速上手

1
建立新專案
點左側「＋ 新增專案」，填入名稱、狀態、projectTag、開始/結束日期（填日期後自動計算預期進度）
2
新增里程碑
點擊專案開啟 Drawer，點「＋ 新增里程碑」，填入名稱與截止日
3
拆解任務
在里程碑下輸入任務，設定優先度（🔴高/🟡中/🟢低），完成後勾選即更新整體進度
4
連結智慧待辦
任務旁點「→ 待辦」自動在智慧待辦建立關聯待辦，帶入 projectTag；智慧待辦亦可升級為里程碑任務

⚠️
Firebase Web API Key 說明： 前端程式碼中可見的 Firebase API Key 屬公開設計（所有 Firebase 網頁應用均如此），真正的存取控制靠 Firestore Security Rules 實施。GitHub 若出現安全警報，屬誤判，可直接 Dismiss 為 false positive。
常見問題

FAQ

忘記密碼怎麼辦？ ▾
目前無密碼重設頁面。請聯繫 IT 管理員，在 Firebase Console → Authentication 中重設密碼。
Gmail 同步授權失敗？ ▾
需使用個人 Google 帳號（非學校 GSuite 帳號，學校帳號的 OAuth 授權受組織政策限制）。本機測試時需在 localhost 環境開啟頁面，GitHub Pages 正式環境則無此限制。
Gemini 分析一直轉圈？ ▾
確認 Gemini API Key 正確且已在 Google AI Studio 啟用。系統有 30 秒 timeout 保護，逾時會顯示錯誤。若使用學校 Google 帳號建立的 Key，quota 可能為 0，需改用個人帳號。
如何新增可登入的使用者？ ▾
在 Firebase Console → Authentication → Users 頁面，點擊「Add user」填入 Email 與密碼即可。
更新程式後網頁沒變化？ ▾
GitHub Pages 部署約需 1 分鐘。可強制重新整理（Ctrl+Shift+R）清除瀏覽器快取。若仍未更新，確認 git push 是否成功，或在 GitHub repo 的 Actions 頁查看部署狀態。
系統設定在哪裡？ ▾
登入 Portal 後，右上角「⚙ 系統設定」按鈕。可管理各子系統狀態、公告、布告欄、UI 主題。
